Guía Completa para Configurar el Despliegue Automático con Git en Hostinger

Aquí tienes el orden de los pasos y las consideraciones clave para configurar el despliegue automático de tu proyecto en Hostinger, evitando los errores que encontramos:

Paso 1: Preparación del Proyecto Local

  1. Inicializa tu repositorio Git localmente y conecta con GitHub.

    • Asegúrate de que tu repositorio de GitHub sea privado (si es el caso) y que la rama principal se llame main.
    git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/tu_usuario/tu_repositorio.git # O la URL SSH si ya la tienes
git push -u origin main

Paso 2: Configuración en Hostinger (hPanel)

  1. Accede a tu hPanel y ve a la sección Git.

  2. Genera la Clave SSH (CRÍTICO para repositorios privados):

    • Haz clic en "Generar clave SSH".
    • Copia la clave pública que te proporciona Hostinger.

  3. Vacía el Directorio de Despliegue:

    • ¡MUY IMPORTANTE! La carpeta en tu servidor donde se desplegará el código (ej. /home/u400283574/domains/renangalvan.net/public_html/inbox_colibri) debe estar COMPLETAMENTE VACÍA antes de crear el repositorio en Hostinger. Si contiene archivos, Hostinger no te permitirá crearlo.
    • Puedes vaciarla usando FileZilla o SSH (con mucho cuidado con rm -rf * si usas SSH).

  4. Crea el Repositorio en Hostinger:

    • Haz clic en "Añadir repositorio" o similar.
    • URL del Repositorio: Aquí es CRÍTICO.
      • Para repositorios privados: Usa la URL SSH de tu repositorio de GitHub. Ejemplo: git@github.com:tu_usuario/tu_repositorio.git
      • Para repositorios públicos: Puedes usar la URL HTTPS: https://github.com/tu_usuario/tu_repositorio.git
    • Rama (Branch): main (o la rama que uses para despliegue).
    • Directorio de Despliegue: La ruta completa en tu servidor (ej. /home/u400283574/domains/renangalvan.net/public_html/inbox_colibri).
    • Una vez creado, el repositorio debería aparecer en la lista.

Paso 3: Configuración en GitHub

  1. Añade la Clave Pública SSH a GitHub (CRÍTICO para repositorios privados):

    • Ve a tu repositorio en GitHub -> Settings -> Deploy keys.
    • Haz clic en Add deploy key.
    • Pega la clave pública que copiaste de Hostinger.
    • ¡MUY IMPORTANTE! Marca la casilla 'Allow write access'.
    • Haz clic en Add key.

  2. Configura el Webhook de Hostinger en GitHub:

    • Vuelve a la sección Git en Hostinger.
    • Busca la opción de "Implementación automática" (o "Auto Deployment") y actívala.
    • Hostinger te proporcionará una URL de Webhook (ej: https://webhooks.hostinger.com/deploy/TU_ID_UNICO). Cópiala.
    • Ve a tu repositorio en GitHub -> Settings -> Webhooks.
    • Haz clic en Add webhook.
    • Payload URL: Pega la URL del webhook que te dio Hostinger.
    • Content type: application/x-www-form-urlencoded.
    • Secret: Déjalo vacío (el webhook de Hostinger no lo requiere aquí).
    • Events: Selecciona Just the push event.
    • Asegúrate de que Active esté marcado.
    • Haz clic en Add webhook.

Paso 4: Despliegue Inicial y Prueba

  1. Despliegue Manual Inicial desde Hostinger:

    • Vuelve a la sección Git en tu hPanel.
    • Selecciona tu repositorio.
    • Haz clic en el botón "Implementar" (Deploy). Esto hará el primer git pull y descargará tu código al servidor.
    • Verifica la "última salida de compilación" para asegurarte de que fue exitoso.

  2. Prueba del Despliegue Automático:

    • Haz un cambio pequeño y visible en tu proyecto local.
    • Guarda, haz git add ., git commit -m "Probando despliegue automatico" y git push.
    • Espera uno o dos minutos y recarga tu página web. El cambio debería aparecer automáticamente.

Consideraciones Adicionales:

  • Permisos de Usuario en Hostinger: Si eres colaborador y no el dueño de la cuenta, verifica que tu rol tenga permisos para usar la herramienta Git.
  • Limpieza: Una vez que todo funcione, borra los archivos deploy.php y test_exec.php de tu servidor, ya no son necesarios.
  • Depuración: Si el despliegue automático falla, revisa el historial de despliegues en Hostinger y el log del webhook en GitHub para ver los mensajes de error.

Espero que esta guía te sea de gran utilidad para futuros proyectos. ¡Excelente trabajo al depurar todo esto!
